# Review card — design system

Visual previews of the review card's design system: what the tokens look like,
what each module looks like, and how the card lays out across the six data
scenarios the engine can produce.

Generated by `skills/fomo-kernel/tools/design_bundle.py` and uploaded to this
card's Claude Design design-system project
(`a964fd8b-fb74-4c04-8dab-7978794a15dc`). Rebuild with:

```bash
python3 skills/fomo-kernel/tools/design_bundle.py
```

## What lives where

Deliberately not duplicated — duplication is the disease this system was built
to treat, since it forces every change to be made in several places at once:

| Surface | Owns |
|---|---|
| This project | Visual previews: tokens, modules, six scenarios |
| `docs/output-contract.md` | What renders, and in what order (authority) |
| `docs/layout-constraints.md` | For a designer: what is load-bearing, what is negotiable |
| `docs/design-guidelines.md` | **Module contracts and how to add a visual element** without re-tuning its neighbours |

Read `design-guidelines.md` before adding a visual element. Its core idea is
that **modules own slots and elements fill slots**: the curve cell could be
added safely because it fills the metric cell's existing three slots (label,
body, sub) with a line in the body slot instead of a number — it is not a new
module type. Two earlier attempts that *did* introduce a new module (a
full-width strip, then a hero band) each solved the stated problem and created
a new one.

## Contents

- `foundations/` — colour, spacing, type scale, the four emphasis levels
- `components/` — metric grid, curve cell, trade row, verdict panels,
  next step (the single L1), disclosure footnote
- `scenarios/` — six layouts, A through F, each determined by which engine
  modules have data

## Caveat

These previews carry a hand-maintained mirror of the runtime stylesheet
(`.rc2`/`--` here versus `.rc`/`--rc-` in `card_renderer._HTML_WIDGET_CSS`).
That mirror drifts silently. After changing the runtime stylesheet, rebuild and
re-check; see the generator's module docstring for the intended fix.

## Data

Every value shown is synthetic mock data. No real trade records.
